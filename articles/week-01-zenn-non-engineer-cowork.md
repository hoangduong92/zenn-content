# 非エンジニアでもCoworkを使いこなす方法

## 1. はじめに — エンジニアじゃなくてもAIは使える

「AIツールって、プログラミングできないと使えないんでしょ？」

自分も最初はそう思っていました。でも、Claude Coworkを使ってみて気づいたのは、**コードを一行も書かずに、日本語だけで事務作業を自動化できる**ということです。

この記事では、プログラミング経験ゼロの自分が実際にCoworkを使ってみた体験をもとに、非エンジニアでも安心して使える方法をシェアします。

失敗したこと、うまくいったこと、すべて正直に書きます。

## 2. Coworkの仕組みを理解しよう

技術的な詳細は抜きにして、**最低限知っておくべきこと**だけ説明します。

### Cowork = あなたのデスクトップアシスタント

簡単に言うと、Coworkは**あなたのPC上で働く優秀なアシスタント**です。

**イメージ図：**
```
あなた → 日本語で指示 → Cowork → ファイル操作・文書作成 → 完成！
```

[SCREENSHOT: Coworkの動作フロー図]

### セキュリティについて — サンドボックスとは？

Coworkは「サンドボックス」という仕組みで動いています。

**サンドボックス = 砂場**のイメージです。子どもが砂場で遊んでも、砂場の外には影響がないように、Coworkも**指定したフォルダの中だけ**で動作します。

**具体的には：**
- 初回起動時に「作業フォルダ」を選択します
- Coworkはそのフォルダ内のファイルだけを操作できます
- フォルダ外のファイルは一切触れません

[SCREENSHOT: フォルダ選択画面とサンドボックス説明図]

だから、**大事なファイルが入っていない専用フォルダ**を作って、そこで練習するのがおすすめです。

### ファイルアクセスの許可

Coworkがファイルを操作するときは、毎回**確認ダイアログ**が表示されます。

[SCREENSHOT: ファイルアクセス許可ダイアログ]

勝手にファイルを削除されることはないので、安心してください。

## 3. 非エンジニアの自分が最初にやってみた5つのこと

実際に試した手順を、そのまま時系列で紹介します。

### ① Claude Desktopをインストール

公式サイト（https://claude.ai）からダウンロードして、インストーラーを実行するだけです。

**所要時間：** 5分

**注意点：**
- Proプラン ($20/月) の登録が必要です
- 最初の7日間は無料トライアルがありました（2026年2月時点）

[SCREENSHOT: インストール完了画面]

### ② 安全な作業フォルダを選択

デスクトップに「CoworkTest」という名前のフォルダを新規作成しました。

**なぜテストフォルダを作ったか：**
- 大事なファイルを誤って操作されないため
- 気軽に試行錯誤できるため
- いつでも削除してやり直せるため

[SCREENSHOT: デスクトップのCoworkTestフォルダ]

この判断は正解でした。最初は失敗も多かったので。

### ③ 最初のタスク: メモを作成してみた

一番シンプルなタスクから始めました。

**入力したプロンプト：**
```
今日の日付でメモファイルを作成してください。
タイトルは「はじめてのCowork」にしてください。
```

**結果：**
- ファイル名「20260218_はじめてのCowork.txt」が生成されました
- 所要時間：約10秒

[SCREENSHOT: 生成されたメモファイル]

**感想：** 想像以上に簡単！これならできそうと思えました。

### ④ ファイル整理タスク: フォルダを自動整理

次に、少し複雑なタスクに挑戦しました。

**準備：**
- テストフォルダに50個のダミーファイル（画像、PDF、Excel、Word）を配置
- わざとバラバラに配置して、整理タスクを依頼

**入力したプロンプト：**
```
このフォルダのファイルを種類別に整理してください。

【分類ルール】
- 画像ファイル → 「images」フォルダ
- PDFファイル → 「documents」フォルダ
- Excel/Word → 「office」フォルダ
- その他 → 「others」フォルダ

元のファイル名は変更しないでください。
```

**結果：**
- 4つのフォルダが自動作成されました
- 50個のファイルが正しく分類されました
- 所要時間：約2分

[SCREENSHOT: 整理前と整理後のフォルダ比較]

**感想：** 手動でやったら30分以上かかる作業が、一瞬で終わった！

### ⑤ 文書作成タスク: メールを書いてもらう

実用的なタスクとして、メール文面の作成を試しました。

**入力したプロンプト：**
```
取引先の田中様宛に、納期遅延のお詫びメールを作成してください。

【背景】
- 商品：A製品100個
- 当初納期：3月10日
- 新しい納期：3月20日
- 理由：原材料調達の遅れ

【条件】
- 丁寧な敬語で
- 誠意が伝わる文面で
- 今後の対策も記載
```

**結果：**
きちんとしたビジネスメールが完成しました。自分で書くよりも敬語が自然で、構成もしっかりしていました。

**感想：** この精度なら、そのまま使える！

## 4. プロンプトの書き方 — 非エンジニア版

5つのタスクを試してみて、**プロンプトの書き方にコツがある**と気づきました。

### 基本の3原則

✅ **具体的に書く** - 曖昧な指示は曖昧な結果になる
✅ **条件を明記する** - フォーマット、長さ、スタイルを指定する
✅ **文脈を伝える** - 背景や目的を説明すると精度が上がる

### ❌ 悪い例 vs ✅ 良い例

**例1: メール作成**

❌ 悪い例:
```
メールを書いて
```

✅ 良い例:
```
取引先の〇〇様宛に、△△の件でお詫びのメールを作成してください。
丁寧な敬語で、誠意が伝わる文面でお願いします。
```

---

**例2: ファイル整理**

❌ 悪い例:
```
ファイルを整理して
```

✅ 良い例:
```
このフォルダのファイルを種類別にサブフォルダに整理してください。
画像は「images」、文書は「documents」に分類してください。
元のファイル名は変更しないでください。
```

---

**例3: 報告書作成**

❌ 悪い例:
```
報告書を作って
```

✅ 良い例:
```
以下のメモから、上司向けの週次報告書をWord形式で作成してください。
セクションは「今週の成果」「来週の予定」「課題」の3つで、
A4で1ページ以内にまとめてください。
```

### よく使うプロンプトテンプレート

実際に使って便利だったテンプレートを5つ紹介します。

#### テンプレート1: お詫びメール
```
取引先の〇〇様宛に、〇〇の件でお詫びのメールを作成してください。

【背景】
- 問題の内容：〇〇
- 原因：〇〇
- 対応策：〇〇

【条件】
- 丁寧な敬語で
- 誠意が伝わる文面で
- 今後の対策も記載
```

#### テンプレート2: 週報作成
```
以下のメモから、上司向けの週次報告書をWord形式で作成してください。

【メモ】
〇〇

【フォーマット】
- タイトル：週次報告書（〇月〇日〜〇月〇日）
- セクション：今週の成果、来週の予定、課題・懸念事項
- 箇条書き形式
- A4で1ページ以内
```

#### テンプレート3: フォルダ整理
```
このフォルダのファイルを種類別にサブフォルダに整理してください。

【分類ルール】
- 画像ファイル → 「images」フォルダ
- 文書ファイル → 「documents」フォルダ
- Excel/CSV → 「data」フォルダ
- その他 → 「others」フォルダ

【条件】
- 元のファイル名は変更しない
- 空のフォルダは作成しない
```

#### テンプレート4: データ集計
```
このCSVデータを分析して、以下の集計を行ってください。

【集計内容】
- 月別の合計
- カテゴリ別の内訳
- 前月比の計算

【出力】
- Excelファイルで出力
- 集計表とグラフを含める
```

#### テンプレート5: 比較表作成
```
〇〇と△△を比較する表をExcelで作成してください。

【比較項目】
- 価格
- 機能
- サポート
- 評価

【フォーマット】
- 横軸：比較対象
- 縦軸：比較項目
- 総合評価も追加
```

これらは、そのまま〇〇の部分を埋めて使えます。

## 5. よくある失敗と対処法

自分が実際に失敗したことと、その解決方法をシェアします。

### 失敗1: 指示が曖昧すぎた

**やったこと：**
```
資料を作って
```

**結果：**
「どんな資料ですか？」と聞き返されました。

**対処法：**
具体的に書き直しました。
```
〇〇についてのプレゼン資料を、PowerPointで5枚のスライドで作成してください。
構成は「課題」「提案」「効果」「スケジュール」「まとめ」でお願いします。
```

### 失敗2: 作業フォルダを選び忘れた

**やったこと：**
Coworkモードを有効にしただけで、フォルダを指定していませんでした。

**結果：**
「作業フォルダが選択されていません」というエラーが表示されました。

[SCREENSHOT: エラーメッセージ]

**対処法：**
設定画面から作業フォルダを選択し直しました。

### 失敗3: 期待値が高すぎた

**やったこと：**
「このPDFを読んで、完璧な提案書を作って」と依頼しました。

**結果：**
出力された提案書は、構成は良いものの、細かい調整が必要でした。

**学んだこと：**
Coworkは「たたき台を作るツール」として使うのがベスト。最終的な調整は人間が行う前提で使うと、ストレスなく使えます。

## 6. 実際の活用事例3選

1週間使ってみて、実用的だった事例を紹介します。

### 事例1: デスクトップのファイル整理

**状況：**
デスクトップに100個以上のファイルが散らかっていました。

**やったこと：**
```
Desktopフォルダのファイルを整理してください。

【分類ルール】
- 2026年のファイル → 「2026」フォルダ
- 2025年以前 → 「archive」フォルダ
- さらに各フォルダ内で種類別に分類

【条件】
- 日付はファイルの更新日を使用
- ファイル名は変更しない
```

**結果：**
- 100個のファイルが5つのフォルダに整理されました
- 所要時間：約2分（手動なら1時間以上）

[SCREENSHOT: 整理前と整理後のデスクトップ]

**効果：**
デスクトップがスッキリして、必要なファイルがすぐ見つかるようになりました。

### 事例2: 週報の自動生成

**状況：**
毎週金曜日に上司へ週報を提出する必要がありました。

**やったこと：**
1週間分の簡単なメモを用意しました。
```
【メモ】
月: 顧客A社訪問、提案書作成
火: 社内会議2件、見積作成
水: 顧客B社オンライン商談
木: 資料作成、メール対応
金: 週報作成、来週準備
```

そして以下のプロンプトで週報を依頼：
```
以下のメモから、上司向けの週次報告書をWord形式で作成してください。

【メモ】
（上記のメモを貼り付け）

【フォーマット】
- タイトル：週次報告書（2月16日〜2月20日）
- セクション：今週の成果、来週の予定、課題・懸念事項
- 箇条書き形式
- A4で1ページ以内
```

**結果：**
- きちんとした体裁の週報が3分で完成しました
- そのままWordファイルとして保存されました

[SCREENSHOT: 生成された週報]

**効果：**
週報作成時間が30分 → 5分に短縮されました。

### 事例3: 競合調査の比較表作成

**状況：**
競合3社のサービスを比較する必要がありました。

**やったこと：**
```
以下の3社のサービスを比較する表をExcelで作成してください。

【対象】
- A社、B社、C社

【比較項目】
- サービス概要
- 価格
- 機能
- サポート体制
- 評価（5段階）

【フォーマット】
- 横軸：3社
- 縦軸：比較項目
- 見やすい表形式で
```

**結果：**
- 比較表が自動生成されました
- 各社の情報は自分で調べたものを後から入力しました

[SCREENSHOT: 生成された比較表]

**効果：**
Excelのフォーマット作成時間が20分 → 1分に短縮されました。

## 7. まとめ

非エンジニアの自分が1週間Coworkを使ってみて感じたことをまとめます。

### 使えると感じたポイント

✅ **プログラミング知識は本当に不要** - 日本語だけで操作できる
✅ **事務作業の時間が50%以上削減** - ファイル整理、文書作成が劇的に速くなった
✅ **思ったより簡単** - セットアップから実際の利用まで30分で始められた

### 使いこなすコツ

✅ **具体的な指示を心がける** - 曖昧な指示は曖昧な結果になる
✅ **テンプレートを作っておく** - よく使うタスクはプロンプトを保存しておく
✅ **たたき台を作るツールと割り切る** - 最終調整は人間が行う前提で使う

### 注意点

⚠️ **最初は専用フォルダで練習する** - 大事なファイルがあるフォルダは避ける
⚠️ **100%完璧は期待しない** - 8割完成を目指して、残り2割は自分で調整
⚠️ **機密情報の扱いに注意** - セキュリティポリシーを確認してから利用
